## TECHNICAL ARCHITECTURE

### Tech Stack

**Framework:** Next.js 14+ (App Router)
**Database:** Supabase PostgreSQL (or PlanetScale MySQL)
**Hosting:** Vercel Pro ($20/mo)
**CDN:** Cloudflare (free tier)
**Search:** Algolia or Typesense (for verse/topic search)
**Analytics:** Plausible or PostHog
**Email:** Resend or SendGrid
**CMS:** Contentful or custom with MDX

### Database Schema

#### Tables:

**1. verses**
```sql
CREATE TABLE verses (
  id SERIAL PRIMARY KEY,
  book VARCHAR(50) NOT NULL,
  chapter INT NOT NULL,
  verse INT NOT NULL,
  text_kjv TEXT,
  text_niv TEXT,
  text_esv TEXT,
  text_nlt TEXT,
  text_msg TEXT,
  slug VARCHAR(100) UNIQUE, -- e.g., "john-3-16"
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);
CREATE INDEX idx_book_chapter_verse ON verses(book, chapter, verse);
CREATE INDEX idx_slug ON verses(slug);
```

**2. topics**
```sql
CREATE TABLE topics (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,
  slug VARCHAR(100) UNIQUE,
  description TEXT,
  parent_topic_id INT REFERENCES topics(id), -- for sub-topics
  created_at TIMESTAMP DEFAULT NOW()
);
```

**3. verse_topics** (many-to-many)
```sql
CREATE TABLE verse_topics (
  verse_id INT REFERENCES verses(id),
  topic_id INT REFERENCES topics(id),
  relevance_score INT, -- 1-10, for sorting
  PRIMARY KEY (verse_id, topic_id)
);
```

**4. generated_content**
```sql
CREATE TABLE generated_content (
  id SERIAL PRIMARY KEY,
  page_type VARCHAR(50), -- 'verse', 'topic', 'intent', etc.
  slug VARCHAR(200) UNIQUE,
  title VARCHAR(200),
  meta_description VARCHAR(300),
  content_json JSONB, -- structured content sections
  word_count INT,
  published BOOLEAN DEFAULT FALSE,
  generated_at TIMESTAMP DEFAULT NOW(),
  last_updated TIMESTAMP DEFAULT NOW()
);
```

**5. cross_references**
```sql
CREATE TABLE cross_references (
  verse_id INT REFERENCES verses(id),
  related_verse_id INT REFERENCES verses(id),
  relationship_type VARCHAR(50), -- 'same_topic', 'parallel', 'contrast', etc.
  PRIMARY KEY (verse_id, related_verse_id)
);
```

### Rendering Strategy

**Hybrid Approach:**

1. **Pre-render top 1,000 pages (SSG):**
   - Most popular verses (john 3:16, psalm 23, etc.)
   - Main topic hubs (love, faith, hope, etc.)
   - Tool landing pages
   - Homepage, about, contact

2. **ISR for mid-tail (20,000 pages):**
   - Medium-traffic verses
   - Popular sub-topics
   - Common intent pages
   - Revalidate every 24 hours

3. **On-Demand SSR for longtail (179,000 pages):**
   - Less common verses
   - Niche topics
   - Specific combinations
   - Cache at edge for 7 days
   - Build on first request

**Why this works:**
- Fast initial build (<30 min for 1,000 pages)
- No 200k page build time (would be 20+ hours)
- Pages generated on-demand, cached at edge
- Revalidate only popular pages regularly
- Cost-effective (no massive compute for builds)

### Page Generation Pipeline

**Content Generation:**

1. **Verse Pages:**
   - Template-based with variable insertion
   - AI-generated unique intros, applications, prayers (GPT-4 or Claude)
   - Human-reviewed for top 1,000 pages
   - Auto-generated cross-references from database
   - FAQs generated from common search queries

2. **Topic Pages:**
   - Query database for all verses tagged with topic
   - Sort by relevance score
   - Generate intro with AI (reviewed for main hubs)
   - Auto-create sub-topic navigation
   - Downloadable PDFs generated on-demand (Puppeteer)

3. **Intent Pages:**
   - Template + AI-generated application sections
   - Pull relevant verses from multiple topics
   - Interactive tools embedded (React components)

**Content Pipeline:**
```
1. Seed database with verses (31,102 from Bible API)
2. Tag verses with topics (semi-automated: AI suggestions + human review)
3. Generate cross-references (automated + manual curation)
4. Create content templates for each page type
5. Use AI to generate unique sections (intros, applications, prayers)
6. Human QA review for top 5,000 pages
7. Deploy with ISR/on-demand rendering
```

## CONTENT QUALITY STANDARDS

### Minimum Requirements for Every Page

**Length:**
- Verse pages: 2,000+ words
- Topic pages: 3,000+ words
- Intent pages: 2,500+ words
- Sub-topic pages: 1,500+ words
- Book/chapter pages: 1,500+ words

**Structure:**
- Clear H1 (title)
- Logical H2/H3 hierarchy
- Table of contents (for 1,500+ word pages)
- Scannable (bullet points, short paragraphs)
- Images/visuals (at least 1)

**SEO:**
- Unique title tag (50-60 chars)
- Unique meta description (150-160 chars)
- Target keyword in H1, first 100 words, URL
- 2-3 related keywords naturally placed
- Schema markup (Article, FAQPage)

**User Experience:**
- Fast load (<2s LCP)
- Mobile-responsive
- Easy to read (Flesch score 60+)
- Clear CTAs (email signup, bookmark, share)
- No intrusive ads/popups

**Internal Linking:**
- Breadcrumbs
- 5-10 related verse links
- 3-5 related topic links
- Cross-references
- Contextual inline links (5-10)

**Interactive Elements:**
- At least 1 interactive feature per page
- Verse comparison, audio player, or image generator
- Share buttons
- Copy-to-clipboard
- Bookmark/save option

### AI-Generated Content Guidelines

**What AI Generates:**
- Verse explanations (reviewed for theology)
- Introductions to topics
- Application sections
- Prayer prompts
- FAQs (from search query data)

**What Humans Review:**
- All theological content (top 5,000 pages)
- Main topic hub pages (2,000 pages)
- Tool descriptions
- Study guides

**Quality Control:**
- AI content tagged in database
- Random sampling for review (10% monthly)
- User feedback loop (report errors)
- Continuous improvement

### Preventing Duplicate Content

**Strategies:**
1. **Unique angles:** Each page answers different questions
2. **Template variation:** 5+ templates per page type, rotated
3. **AI-generated sections:** Always unique
4. **Cross-references:** Different for each verse/topic
5. **FAQs:** Based on actual search queries, unique per page
6. **Examples:** Real-world applications vary

**Cannibalization Prevention:**
- Clear keyword targeting (1 primary per page)
- Different search intent for similar topics
- Canonical tags where needed
- Topic clusters properly structured

## INTERNAL LINKING STRATEGY

### Hub-and-Spoke Model (3 Levels)

**Level 1: Main Topic Hubs (100 pages)**
- Examples: Love, Faith, Hope, Strength, Forgiveness, Healing
- URL: `/verses-about/love`
- Links out to:
  - 5-10 sub-topic pages (Level 2)
  - Top 10 verses for this topic (Level 3)
  - Related main topics (Level 1)

**Level 2: Sub-Topic Pages (2,000 pages)**
- Examples: Romantic Love, God's Love, Self-Love, Love for Enemies
- URL: `/verses-about/love/romantic-love`
- Links out to:
  - 10-30 individual verses (Level 3)
  - Related sub-topics (Level 2)
- Links up to:
  - Parent topic hub (Level 1)

**Level 3: Verse Pages (31,102 pages)**
- Examples: John 3:16, 1 Corinthians 13:4
- URL: `/verse/john/3/16`
- Links up to:
  - 3-5 relevant topics (Level 1 or 2)
- Links across to:
  - 5-10 cross-reference verses (Level 3)
  - Previous/next verse in chapter

### Every Page Has:

**1. Breadcrumbs**
```
Home > Verses About Love > Romantic Love > 1 Corinthians 13:4
```

**2. Related Verses Sidebar (5-10 links)**
- Algorithmically generated (same topics, similar themes)
- Manually curated for top pages
- Randomized to distribute link equity

**3. Popular in This Category (3-5 links)**
- Top verses for the topic
- Changes based on page type

**4. Cross-References Section (5-10 links)**
- Biblically related verses
- Pulled from database table

**5. Topic Tags (3-7 links)**
- Link to topic hub pages
- Consistent across verses on same topic

**6. Contextual Inline Links (5-10)**
- Within body content
- Natural anchor text
- Link to related verses, topics, tools

### Link Distribution Strategy

**Priority:**
1. Internal links > external links (ratio 20:1)
2. Deep links (to longtail pages) > homepage links
3. Contextual > sidebar > footer
4. Varied anchor text (avoid over-optimization)

**Automation:**
- Build link suggestion algorithm
- Auto-generate related verses based on topics
- Manual curation for top 1,000 pages
